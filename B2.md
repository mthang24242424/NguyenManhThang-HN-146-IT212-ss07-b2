# BÀI 2: Tối ưu Prompt (Thực hành viết Prompt giải nghĩa Stack Trace và gỡ lỗi)

## 1. Prompt sau khi tối ưu

```text
Hãy đóng vai trò là một Java Debugger Senior/Java Tech Lead có nhiều kinh nghiệm phân tích Stack Trace và xử lý lỗi trong các hệ thống Spring Boot.

[Ngữ cảnh]

Tôi đang chạy thử lớp PaymentProcessor và gặp lỗi NullPointerException.

Mã nguồn:

import java.util.Map;

public class PaymentProcessor {

    private Map<String, Double> exchangeRates;

    public double processPayment(String currency, double amount) {
        double rate = exchangeRates.get(currency);
        return amount * rate;
    }
}

Stack Trace thực tế:

Exception in thread "main"
java.lang.NullPointerException:
Cannot invoke "java.util.Map.get(Object)"
because "this.exchangeRates" is null

at PaymentProcessor.processPayment(PaymentProcessor.java:7)
at Main.main(Main.java:8)

[Mục tiêu]

Phân tích chi tiết lỗi dựa trên Stack Trace và mã nguồn.

[Yêu cầu xử lý]

1. Xác định chính xác Root Cause (nguyên nhân gốc rễ).
2. Giải thích từng dòng quan trọng trong Stack Trace.
3. Chỉ rõ dòng code gây lỗi.
4. Đề xuất phương án sửa chữa an toàn nhất.
5. Khởi tạo exchangeRates bằng ConcurrentHashMap.
6. Không làm mất tính đóng gói (Encapsulation) của OOP.
7. Không chuyển exchangeRates thành public.
8. Đề xuất các biện pháp phòng tránh lỗi tương tự trong tương lai.

[Đầu ra mong muốn]

Phần 1: Phân tích Root Cause.

Phần 2: Giải thích Stack Trace.

Phần 3: Mã nguồn Java đã sửa hoàn chỉnh.

Phần 4: Các khuyến nghị về thiết kế OOP và phòng tránh NullPointerException.
```

# 2. Mã nguồn Java đã sửa hoàn chỉnh

```java
import java.util.Map;
import java.util.concurrent.ConcurrentHashMap;

public class PaymentProcessor {

    private final Map<String, Double> exchangeRates;

    public PaymentProcessor() {
        this.exchangeRates = new ConcurrentHashMap<>();
    }

    public void addExchangeRate(String currency, double rate) {
        exchangeRates.put(currency, rate);
    }

    public double processPayment(String currency, double amount) {

        Double rate = exchangeRates.get(currency);

        if (rate == null) {
            throw new IllegalArgumentException(
                "Exchange rate not found for currency: " + currency
            );
        }

        return amount * rate;
    }
}
```

# 3. Giải thích lỗi do AI phân tích

## Root Cause

Nguyên nhân gốc rễ là biến:

```java
exchangeRates
```

được khai báo nhưng chưa được khởi tạo.

```java
private Map<String, Double> exchangeRates;
```

Khi đối tượng PaymentProcessor được tạo ra, biến này mặc định nhận giá trị:

```java
null
```

Sau đó chương trình thực hiện:

```java
exchangeRates.get(currency);
```

tương đương:

```java
null.get(currency);
```

nên JVM phát sinh:

```java
NullPointerException
```

---

## Phân tích Stack Trace

### Dòng 1

```text
Exception in thread "main"
```

Lỗi xuất hiện trên luồng chính của chương trình.

---

### Dòng 2

```text
java.lang.NullPointerException
```

Chương trình cố truy cập một đối tượng đang mang giá trị null.

---

### Dòng 3

```text
Cannot invoke "java.util.Map.get(Object)"
because "this.exchangeRates" is null
```

JVM chỉ rõ:

* Phương thức bị gọi:

```java
Map.get()
```

* Đối tượng bị null:

```java
this.exchangeRates
```

---

### Dòng 4

```text
at PaymentProcessor.processPayment(PaymentProcessor.java:7)
```

Lỗi xảy ra tại:

```java
double rate = exchangeRates.get(currency);
```

ở dòng 7 của PaymentProcessor.

---

### Dòng 5

```text
at Main.main(Main.java:8)
```

Lớp Main gọi tới:

```java
processPayment()
```

và làm phát sinh lỗi.

# 4. Vì sao ConcurrentHashMap là lựa chọn phù hợp

```java
private final Map<String, Double> exchangeRates
        = new ConcurrentHashMap<>();
```

Ưu điểm:

* Thread-safe.
* Phù hợp môi trường Spring Boot nhiều request đồng thời.
* Tránh lỗi race condition khi đọc/ghi tỷ giá.
* Hiệu năng tốt hơn Hashtable.
* Không cần synchronized toàn bộ phương thức.

---

# 5. Đảm bảo Encapsulation

Biến exchangeRates vẫn được khai báo:

```java
private final Map<String, Double> exchangeRates;
```

Không cho phép truy cập trực tiếp từ bên ngoài.

Các thao tác đều thông qua phương thức:

```java
addExchangeRate(...)
processPayment(...)
```

Điều này đảm bảo nguyên tắc:

* Data Hiding
* Encapsulation
* Maintainability

trong lập trình hướng đối tượng.

# Kết luận

Prompt tối ưu đã áp dụng đầy đủ 5 thành phần:

1. Role (Java Debugger / Tech Lead)
2. Context (Mã nguồn + Stack Trace)
3. Task (Phân tích và sửa lỗi)
4. Constraints (ConcurrentHashMap, Encapsulation)
5. Output Format (Root Cause, Stack Trace, Code, Khuyến nghị)

Nhờ đó AI có thể đưa ra lời giải chính xác, an toàn và sát với yêu cầu thực tế hơn nhiều so với prompt thô ban đầu.
